##Вступление
В качестве индивидуальных правок для предыдущего проекта я решил кардинально переделать структуру кода.
До этого в проекте были модули: main.py, preprocess.py, visualization.py, visualization_advanced.py. После моих правок в проекте остались только main и visualization_advanced.py.
Сейчас я расскажу почему я так сделал. Main.py стал точкой входа, теперь программа запускается через этот модуль. Также preprocess теперь интегрирован именно в этот модуль, тоесть предобработка проходит через него.
Также я решил перевести visualization в visualization_advanced, потому что продвинутая визуализация исключает необходимость в обычной визуализации.

## Что делает программа
При запуске main.py выполняется цепочка действий:
- Чтение melb_data.csv
- Приведение типов (даты/числа), заполнение пропусков
- Добавление новых признаков (например, `SaleYear`, `HouseAge`, `PricePerRoom`, `BuildRatio`, `Density`, `PricePerM2`)
- Сохранение таблиц-агрегатов по районам/регионам в CSV
- Запуск генерации отчёта через `export_analysis_report(...)` из `visualization_advanced.py`
